## 개요
이 문서는 Python Service 개발 시 공통적으로 적용되는 아키텍처 규정에 대해 설명합니다.

## 세부 지침
- 함수형 프로그래밍 패러다임을 따르며, 각 함수는 명확한 입력과 출력을 가지게 함수를 설계하라.
- class export 패턴 보다 함수 export 패턴을 우선적으로 사용하라. 객체지향성 상태관리 패턴 보다 함수 결과를 주입하는 함수형 패턴을 우선적으로 사용하라.
- 필요한 상태(예를 들어 requests.session) 같은 것들은 main.py 진입점에서 생성하여 함수 인자를 통해 전파한다.
- API Response, 명확한 DTO 구조의 함수 Result 전달이 필요한 경우에 한해서 예외적으로 models 디렉터리에 dataclass를 정의할 수 있다.
